- [ ] Task 1: 搭建桌面应用基础框架与单页仪表盘
  - [ ] 选择并初始化桌面端技术栈，优先采用轻量级桌面壳与 Web UI 组合
  - [ ] 搭建单页仪表盘骨架，包含目录选择、规则区、统计区、扫描结果区、计划任务区
  - [ ] 明确本地文件系统访问、删除能力和设置持久化方式

- [ ] Task 2: 实现目标文件夹发现与基础统计
  - [ ] 支持从根目录按精确文件夹名递归发现目标文件夹
  - [ ] 支持自定义扩展名白名单过滤
  - [ ] 统计总文件数、匹配文件数、命中文件夹数并在界面展示

- [ ] Task 3: 实现镜头组识别与版本保留规则
  - [ ] 从文件名中解析镜头号、日期和 take 信息
  - [ ] 在每个匹配文件夹内按镜头号分组
  - [ ] 对同镜头组内同日期或同版本点的多个 take 仅保留最新 take
  - [ ] 对归并后的有效版本按修改时间执行“最早 + 最新 + 中间均匀抽样”的稀疏保留
  - [ ] 支持每个镜头组最大保留数量配置

- [ ] Task 4: 实现扫描预览与批量清理执行
  - [ ] 生成待保留文件、待清理文件、预计释放空间的预览结果

  - [ ] 提供“先预览后确认”和“自动执行”两种任务执行模式

- [ ] Task 5: 实现删除策略与回收站提醒
  - [ ] 默认将文件移入回收站
  - [ ] 支持切换为彻底删除并提供风险提示
  - [ ] 记录经由本工具移入回收站的累计空间
  - [ ] 提供定期提醒用户清理回收站的能力

- [ ] Task 6: 实现应用内定期任务
  - [ ] 支持创建、编辑、启停定期扫描任务
  - [ ] 支持设置执行周期和任务执行模式
  - [ ] 记录任务最近执行时间、执行结果和异常信息

- [ ] Task 7: 验证关键流程并完善可用性
  - [ ] 验证 `unity` 精确命名目录发现是否符合预期
  - [ ] 验证镜头组内 take 归并和稀疏保留结果是否正确
  - [ ] 验证预览、回收站删除、彻底删除、计划任务执行链路
